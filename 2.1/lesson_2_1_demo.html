2.1.1 为什么要学流程？

常见痛点

  • 需求不清，开发时不断返工。
  • 前后端接口说不拢，联调崩溃。
  • 上线流程混乱，无法快速回滚。

流程视角能做什么？

  • 每一步都有明确输入 / 输出。
  • 方便引入 Codex / Claude 做协作。
  • 能在 PR、需求讨论中清楚表达。

课堂策略

2.1.2 全流程地图：最小可行链路

1. 需求确认

访谈、PRD、验收标准。

2. 技术选型

框架、数据库、第三方服务。

3. 架构设计

系统边界、接口、数据流。

4. 开发协作

Git 流程、代码规范。

5. 上线发布

CI/CD、灰度、回滚。

6. 运维监控

日志、告警、容量规划。

7. 版本更新

复盘、Roadmap、Patch。

👉 每个节点都能安排一个“AI 助手任务”。

2.1.3 环节详解

需求确认

输出 PRD、用户画像、验收标准。可以请 Codex 帮忙润色 PRD、生成测试用例。

技术选型

列出约束（预算 / 团队 / 性能），比较框架优劣。Claude Code 擅长做决策支持。

架构设计

产出系统拓扑、数据库 ER 图、接口清单。把草图交给 Agent，让它生成文字说明。

开发协作

分支策略、Commit 规范、Code Review。Codex 可充当“虚拟实习生”帮你改代码。

上线 + 运维

CI 配置、部署脚本、监控指标、应急预案。

版本更新

复盘成功 / 失败点，整理下一版 Roadmap。

2.1.4 前后端架构是什么？

餐厅类比

  • 前端 = 服务员：负责菜单、点菜界面。
  • 后端 = 厨房：处理逻辑、访问数据库。
  • API = 传菜口：负责信息交接。

你现在知道“点菜慢”的原因：接口还没准备好。

2.1.5 RESTful API 核心

GET   /api/v1/orders       # 获取订单列表
POST  /api/v1/orders       # 创建订单
GET   /api/v1/orders/{id}  # 查看详情
PATCH /api/v1/orders/{id}  # 局部更新
DELETE /api/v1/orders/{id} # 删除

2.1.6 CORS 与安全

CORS 是什么？

浏览器出于安全，只允许同源请求；跨域访问需要服务器声明白名单。

常见解决方案

  • 后端配置允许域名列表。
  • 使用反向代理（Nginx）转发。
  • 本地开发使用代理 / --proxy 参数。

安全加固

  • Token / Cookie 鉴权。
  • 速率限制、输入校验。
  • CI 中自动跑安全扫描。

用 Codex 写 CORS 中间件，是最快的提效方式之一。

2.1.7 Codex 辅助接口调试

让 Codex 帮你写 API 请求、分析响应、生成文档。

课堂流程

  1. 描述接口目标（URL / 方法 / 参数）。
  2. 请 Codex 生成 cURL 与 fetch 两种写法。
  3. 让它解释响应字段、潜在错误。
  4. 整理成 Markdown API 文档。

输入模板

“帮我调用 POST /api/v1/bookings，参数：name、email、date、packageType。输出 cURL + fetch + 错误处理建议。”
