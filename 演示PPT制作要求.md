# 演示PPT制作要求方案
## 设定
你将会收到一份涵盖知识点的“教案”
这份教案是同时给教师和学生使用的（本质为知识点提纲）
你需要根据这个教案中的内容，生成一份辅助老师讲课和学生理解的网页PPT

## 制作要求
1. **统一模版**  
   - 必须基于 `demos/course_template/` 下提供的 HTML/CSS/JS 模版结构来制作，保持顶部导航、进度条、滑动结构与实验室组件的统一体验。  
   - 只允许在模版约定的占位区域内填充内容或扩展组件，避免另起一套样式体系。

2. **美观性与风格**  
   - 页面需保持现代感与教学友好并重：使用模版预设的色彩体系、留白与阴影，不另行叠加冲突样式。  
   - 每个幻灯片内的卡片/面板/时间线需保持排版对齐，图文比例合理，避免大段文字堆叠。  
   - 交互实验室要保证边框、圆角、背景与模版统一，输入区域和预览区域之间保持视觉分区。

3. **紧贴教案内容**  
   - 必须完整覆盖教案列出的所有知识点：每条知识点都需在 PPT 中有对应的页或模块，确保教师无需额外查阅即可按照教案逐章讲授。  
   - 所有幻灯片的标题、要点、任务、示例代码必须逐条映射教案内容，禁止新增未提及的主题或删减原有重点。  
   - 若教案给出步骤/任务，需在 PPT 中还原“目标—概念—示例—操作—价值”链路，并标明与教案段落的对应关系（可用注释或小标签）。

4. **字体与可读性**  
   - 标题层级：大标题 ≥ 38px，章节标题 28–32px，正文 18–20px，列表/注释 16px，不得低于 16px。  
   - 代码区使用等宽字体（模版默认 JetBrains Mono），字号保持 15–16px，并提供足够行高（≥1.5）。  
   - 确保在 1080p 投屏距离 3 米内可清晰阅读。

5. **例题与交互**  
   - 每个例题或练习必须采用交互式呈现（如代码实验室、点击揭示、步骤提示），引导学生先思考/动手，再获得提示。  
   - 禁止直接展示最终答案；应提供任务说明、必要提示、执行按钮或交互步骤，让学习者在操作中得出结论。  

6. **图片与素材存放**  
   - 图片文件必须与对应的 HTML 文件处于同一目录（例如将图片复制到课程 demo 目录内，引用相对路径 `./image.png`）。  
   - 禁止引用本地绝对路径或远程外链，确保打包或离线访问时素材完整。  
   - 所有图片需提供 `alt` 描述，并在附近注明用途或来源（如“星光摄影流程示意”）。
